这篇文章给自己复习 PID 用。重点不是背公式，而是把它放进 STM32 的真实链路里：传感器采样 -> 反馈值 -> PID -> PWM/执行器 -> 下一次采样。先把单环跑稳，再理解速度环、位置环、电流环为什么要串起来。

一、PID 在控制什么

PID 的输入只有两类：

  • target：目标值，例如目标速度、目标角度、目标温度。
  • feedback：测量值，例如编码器速度、IMU 角度、NTC 温度。

三个项的物理意义：

项 作用 过大时的常见表现
P 立即根据当前误差动作 抖动、振荡、噪声被放大
I 消除长期静差 积分饱和、超调、恢复慢
D 预测误差变化、增加阻尼 对噪声敏感、输出尖峰

PID 不是“参数越多越好”。大多数电机控制先从 P 开始，速度环通常使用 PI，D 项必须在噪声和采样质量足够好时才考虑。

三、工程增强 PID：积分、微分、偏置与死区

基础 PID 适合验证闭环方向和初步调参；在实际电机、温控、平衡机构里，通常还要加入积分保护、微分滤波、输出偏置和输入死区。

1. 积分限幅、积分分离和变速积分

  • 积分限幅：直接限制积分累计量，防止无限增长。
  • 积分分离：误差很大时不积分。系统离目标很远时优先靠 P 拉回，避免靠近目标后积分过多造成超调。
  • 变速积分：误差小时全速积分；误差变大时逐渐减小积分速度；误差超过阈值后停止积分。它比开/关式积分分离更平滑。

2. 微分先行和不完全微分

普通微分对误差求导，目标值突然阶跃时会产生 D 项尖峰。微分先行改为对测量值求导：

3. 输出偏置和输入死区

  • 输出偏置：在 PID 输出上加一个有明确物理来源的基础驱动力，例如克服静摩擦或平衡机构重力前馈。
  • 输入死区：误差很小时按零处理，避免编码器量化或 ADC 抖动让 PWM 来回跳。死区过大则会留下稳态误差。

参数必须满足 0 <= integral_full_error < integral_stop_error，d_filter_tau_s >= 0。使能控制、故障恢复、切换目标源时调用 PidAdvanced_Reset(&pid, current_feedback)，防止旧积分和首次微分尖峰带入新状态。死区、积分阈值和 dt_s 都要带明确单位，不能使用脱离物理意义的魔法数。

4. 现象与处理的对应关系

现象 优先尝试
输出顶在上限，解除后仍冲很远 积分限幅 + 积分分离
靠近目标时超调明显 变速积分，减小 Ki
改目标时 D 项很大 微分先行
编码器/IMU 噪声让 D 项跳 不完全微分、改善采样或降低 Kd

四、为什么需要多环/串级控制

一个环同时追位置和抗负载扰动往往很难调。串级控制把快变量放内环、慢变量放外环：

  • 电流环：最快，控制转矩，直接面对电机电气动态。
  • 速度环：中等速度，控制转速，抵抗负载变化。
  • 位置环：最慢，输出合理的速度目标，避免一下子把电机打满。

多环的关键不是“多套 PID”，而是带宽分层：内环必须明显快于外环。一个常见参考是电流环几 kHz 到十几 kHz，速度环约 0.5 到 2 kHz，位置环几十到几百 Hz。具体频率取决于电机、驱动器、ADC、编码器和 MCU 的实际性能，必须测量验证。

五、双环例子：速度外环 + 电流内环

速度外环输出的不是 PWM，而是“目标电流”。电流内环再根据实际相电流或母线电流输出 PWM。

在真实电流环中，ADC 采样时刻要尽量与 PWM 同步，避开开关尖峰；需要硬件过流保护和软件限流双重保护。不要只依赖 PID 输出限幅来保护功率器件。

七、单环和多环怎么调

调参顺序必须从内到外：

  1. 只有电流环时，先关闭速度/位置环，确认电流采样方向、零偏、限流和 PWM 方向正确。
  2. 调电流环 P，再加入小 I，观察阶跃响应和过流保护。
  3. 电流环稳定后调速度环，先 P 后 I，速度反馈建议同时记录原始值和滤波值。
  4. 最后调位置环，位置环一般不需要很强的 I，先限制最大速度和最大电流。
  5. 每一层都确认输出限幅、积分限幅和故障退出，再打开下一层。

测试时通过 UART、CAN 或上位机记录：目标值、反馈值、误差、积分项、PID 输出、电流和 PWM。没有曲线，只靠肉眼和感觉调 PID 很容易误判。

八、优化与改进的实际步骤

PID 优化不是不断试 Kp/Ki/Kd，而是一个能回退、能对比的工程流程。每次只改一个变量，并保留上一次可工作的参数。

  D --> E[逐层打开外环]
  E --> F[阶跃与负载扰动测试]
  F --> G[记录参数与回归验证]
  G --> H[固化版本]

1. 建立基线，不要一上来就改 PID

先关闭积分和微分，只给一个很小的固定 PWM，记录：供电电压、空载电流、编码器速度、方向、采样周期和机械噪声。这样能区分“硬件问题”和“PID 参数问题”。如果电机空载都抖、反馈方向不对或速度值跳变，先修传感器、电源、编码器和驱动，再谈 PID。

2. 明确每层的输入、输出和单位

给每个变量写单位，例如位置用 rad、速度用 rps、电流用 A、PWM 用计数值或百分比。检查外环输出是否正好是内环目标：位置环输出速度目标，速度环输出电流目标，电流环输出 PWM。单位混乱是串级控制最隐蔽的错误之一。

3. 固定时序并测量耗时

在 STM32 上用定时器固定频率触发控制；通过 GPIO 翻转或 DWT 周期计数器测量控制函数耗时。控制计算必须明显小于周期，例如 1 kHz 环路周期为 1 ms，就不应让 ISR 接近 1 ms。ADC 采样、DMA 更新和 PWM 更新的时序要可重复。

4. 先调 P，再逐步加入 I

从很小的 Kp 开始增加，直到响应足够快但未持续振荡；再缓慢增加 Ki 消除静差。每次参数修改都做同样幅度的目标阶跃，例如速度从 0 rps 到一个安全目标值，并记录上升时间、超调、稳态误差和恢复时间。

5. 在受控条件下测试扰动

对速度环，可在恒定目标下轻微增加机械负载；对位置环，可施加小扰动后释放。观察系统是否恢复、是否长时间饱和、积分是否过大。不要在没有电流限制、机械限位或急停的情况下做大阶跃测试。

6. 只在有证据时增加滤波或 D

反馈噪声确实影响输出时，先检查接线、接地、编码器计算和采样同步；随后再使用低通或卡尔曼滤波。滤波会引入延迟，D 项也会放大噪声。引入它们后，需要重新检查相位滞后和 PID 稳定性，不能把“更平滑”直接等同于“更好”。

7. 保存可回退的参数版本

将参数、控制频率、限幅、电机型号、编码器线数、供电条件和测试结果一起保存到 Flash 配置、上位机文件或版本库。参数脱离硬件条件没有可比性。每次改动后做一次基本回归：正反转、目标阶跃、失能、急停、传感器断开和过流保护。

九、常见坑

1. 采样周期不固定

PID 的 dt_s 必须与实际控制周期一致。把 1 ms 写成 1.0f 而不是 0.001f，积分和微分都会差 1000 倍。用定时器中断驱动控制环；若调度存在抖动，应测量并限制异常 dt。

2. 正负方向不统一

电机正转、编码器正计数、速度正值、PWM 正方向必须一致。若反馈方向反了，PID 会越调越偏，表现为上电立刻冲到限幅。首次测试先用很小输出，并准备急停。

3. 积分饱和

执行器已经到最大 PWM 或电流限制时，误差仍存在，积分会不断累计。解除限制后会发生严重超调。至少做积分限幅；更进一步可在输出饱和且误差继续推动饱和时暂停积分。

4. 微分直接对误差

目标值突然阶跃时，误差微分会产生尖峰，称为 derivative kick。工程上常对测量值求导，或给目标值加斜坡；噪声大的编码器/IMU 也不宜直接上大 D。

5. 外环比内环快

如果速度环每次都在内环还没稳定时大幅改变目标电流，系统容易振荡。保持内环快、外环慢，并给外环输出加限幅和斜坡。

6. 在中断里做阻塞操作

高频控制中断里不要 printf、等待 ADC、访问阻塞 I2C 或调用可能耗时的函数。采样应由 DMA/定时器安排，控制中断只读取最新数据、计算 PID、更新寄存器。

7. 忽略功率和安全

电机堵转电流、驱动器温升、电源压降、反接、急停和机械限位都不是 PID 能解决的问题。限流、硬件过流关断、失控超时和故障默认关 PWM 必须独立存在。

十、最小复习清单

  • 先确认传感器零偏、比例、单位和正负方向。
  • 单环先跑 P，再加 I，D 最后考虑。
  • 多环从内到外调，内环频率更高。
  • 每个 PID 都要有输出限幅和积分保护。
  • 控制中断固定周期，耗时操作移到低优先级任务。
  • 用曲线和日志调参，别一次改多个参数。
