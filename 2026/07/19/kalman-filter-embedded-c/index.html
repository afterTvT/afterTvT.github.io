这篇文章给自己复习用。目标是把卡尔曼滤波放回实际嵌入式链路里理解：传感器采样有噪声，MCU 先估计可信状态，再把状态交给 PID 或后续逻辑。文中的代码采用单变量、离散时间模型，适合 STM32 上的 ADC、电机速度、IMU 单轴角度等场景。

一、先说人话：卡尔曼滤波到底解决什么

传感器读到的不是“真实值”，而是“真实值加噪声”。例如编码器速度会因量化、抖动出现跳变；加速度计角度在震动时不稳定；ADC 会有电源纹波和采样噪声。

最简单的处理是滑动平均，但它只会平均历史数据，响应会变慢。卡尔曼滤波的想法更进一步：

  1. 根据上一时刻状态和系统规律，先预测当前值。
  2. 读取当前传感器测量值。
  3. 根据“模型可信度”和“测量可信度”计算权重。
  4. 得到这一次更可信的状态估计。

它不是神奇地消灭噪声，而是在“相信预测”和“相信测量”之间做可计算的折中。

flowchart LR
  A[传感器采样] --> B[标定与有效性检查]
  B --> C[卡尔曼预测]
  C --> D[卡尔曼测量更新]
  D --> E[状态估计值]
  E --> F[PID 或阈值决策]
  F --> G[PWM / 执行器]

三、它从哪里来：预测和测量的两种不确定性

离散线性系统常写成：

  • x(k)：系统真实状态，例如“真实速度”或“真实角度”。
  • u(k)：控制输入，例如电机占空比或角速度。
  • z(k)：传感器测量值。
  • A：状态如何自然演化。
  • B：输入对状态的影响。
  • H：状态如何映射到传感器。
  • w(k)：过程噪声，代表模型不完美、外部扰动。
  • v(k)：测量噪声，代表传感器误差。

在单变量入门场景中，可先取 A = 1、H = 1。若状态是速度且没有可靠动力学模型，预测就先认为“下一时刻速度接近上一时刻速度”；如果状态是角度、已知陀螺仪角速度，则用角速度积分做预测。

卡尔曼滤波保存两样东西：

  • x：当前状态估计值。
  • P：对当前估计的不确定度，P 越大代表越没把握。

四、一维离散卡尔曼公式

1. 预测阶段

Q 是过程噪声强度。系统受扰动越明显、模型越粗糙，Q 应越大。预测以后不确定度会上升，因此 P_pred 增大。

这里的代码约定 Q 是“每秒增加多少不确定度”，所以写成 Q * dt。这不是所有模型通用的唯一写法：若把 Q 定义为角速度噪声方差，积分到角度后的项可能与 dt^2 有关；多状态模型则使用矩阵 Q。先写清 Q 的物理定义和单位，再讨论公式，不能直接照搬别人的数值。

2. 测量更新阶段

R 是测量噪声方差。K 是卡尔曼增益，范围通常在 0 到 1：

  • R 大：测量不可靠，K 小，更信任预测。
  • Q 大或长时间没测量：P_pred 大，K 大，更信任新测量。

这些式子来自高斯概率分布的递推：预测给出一个“先验估计”，测量给出一个“观测信息”，两者按各自不确定度加权后得到“后验估计”。工程里先理解这个权重关系，比死记矩阵推导更有用。

八、Q、R、P 到底怎么调

不要先照搬别人的数字。下面是比较可靠的调参步骤：

  1. 让设备静止，连续记录一段原始测量值，计算或观察波动范围。
  2. 用波动的方差作为 R 的起点；数据不够时先取一个相对较大的 R，避免过度相信传感器。
  3. 从较小的 Q 开始。响应太慢时逐步增加 Q；输出仍然抖时，检查 R 是否过小。
  4. initial_p 表示启动时有多不确定。开机就知道初值时可以小一些，开机状态未知时可大一些。
  5. 每次只修改一个参数，同时记录原始值、滤波值和控制输出，方便回看。

例如速度反馈突然跟不上真实变化，优先怀疑 Q 太小或控制周期 dt 不正确；滤波值追着噪声跑，优先怀疑 R 太小。

九、工业/嵌入式场景的常见踩坑

1. dt 不稳定或单位错了

dt 写成毫秒却当秒用，会让积分量差 1000 倍。定时器中断应使用固定周期；若周期可能抖动，应读取真实时间戳计算 dt_s，并限制异常的大 dt。

2. 量测没有先标定

滤波不能修正固定偏置。陀螺仪零偏、ADC 零点、编码器方向和比例系数必须先校准；否则滤波器会很平滑地输出一个错误值。

3. 把异常值当正常测量更新

通信丢帧、ADC 饱和、编码器跳变都可能产生离群点。更新前可做范围检查：

更严格时可以判断创新量 measurement - kf.x 是否超过合理阈值，超出则拒绝本次更新或暂时增大 R。

4. r 为 0 或协方差发散

R = 0 会造成分母风险，也等于宣称传感器绝对准确，工程上不成立。P 变成 NaN 或无限大时，先检查 dt、Q/R 是否为负、输入是否为 NaN，以及是否发生除零。

5. 在中断中做太重的事

一维滤波计算量很小，适合放在 1 kHz 控制中断；但串口打印、浮点格式化、动态分配和复杂矩阵求逆不适合放进高频中断。多维滤波应先测量执行时间，确认不会挤占 PWM、编码器和通信时序。

6. 多轴问题硬套一维模型

三轴姿态、位置速度联合估计需要考虑状态耦合，不能简单把“每个轴各滤一次”当作完整姿态解算。入门时先从每个传感器独立的一维估计做起，确实需要耦合时再使用扩展卡尔曼滤波（EKF）或专门姿态算法。

十、初始化、丢帧与异常量测处理

滤波器不是“初始化一次就永远可靠”。启动、传感器断线、模式切换和异常测量都需要明确策略。

  • 启动初始化：静止启动时，可用多次测量平均值作为 initial_x；已知初值可信时设小 initial_p，未知时设大一些，让滤波器更快接受测量。
  • 丢帧/无效测量：仍执行预测，但跳过更新。此时 P 会增大，表达“随着时间过去我越来越没把握”。
  • 恢复测量：先做范围和创新量检查，不要把恢复第一帧直接当绝对可信数据。
  • 模式切换/急停：若状态物理意义发生突变，应重新初始化或显式增大 P，不要带着旧状态硬算。

十三、卡尔曼滤波的短板与工程处理策略

卡尔曼滤波不是“加上就更准”。它依赖模型、时间、传感器和参数都基本可信；现场条件不满足时，滤波器可能稳定地输出错误结果。工程上先解决数据链路和保护，再考虑把一维模型升级成 EKF/UKF。

1. 模型不准：滤波器不知道真实系统发生了什么

一维速度模型常假设“速度下一刻接近上一刻”，但电机突然加负载、供电电压下降或机构碰撞时，这个假设会失效。表现为滤波值明显滞后或长期偏离真实值。

处理顺序：

  1. 记录输入、测量和估计值，确认偏差是模型问题而不是传感器接线/标定问题。
  2. 在扰动较大时适当增大 Q，让滤波器更快接受新测量。
  3. 若偏差来自一个长期存在的物理量，例如陀螺仪零偏，就把它加入状态，而不是无限增大 Q。
  4. 只有系统确实非线性、线性化误差明显时，再评估 EKF/UKF；它们增加的是模型能力，也增加了实现、调试和算力成本。

2. 测量异常：离群点、丢帧和饱和会污染状态

编码器毛刺、ADC 饱和、I2C 读错、UART 帧错位和 IMU 强震动都会产生异常测量。标准更新公式默认测量服从高斯噪声，不能自动识别这种异常。

工程做法：

  • 每个测量带有效标志：通信 CRC、ADC 饱和位、传感器数据就绪位和范围检查。
  • 使用创新量门限，超限时跳过更新或临时增大 R。
  • 丢帧时继续预测，但设置最大连续丢帧计数；超过上限进入降级或停机状态，不要无限期用旧状态控制电机。
  • 对安全相关控制，异常检测必须独立于滤波器，例如硬件过流、限位开关和驱动器故障脚不能被滤波结果掩盖。

3. 时间不同步：错一个采样时刻，滤波再漂亮也没用

卡尔曼预测依赖 dt。如果编码器速度是 1 ms 前的、IMU 是当前的、ADC 又是 DMA 缓冲区的旧数据，却在同一次更新里当作同一时刻，会产生难以解释的相位差和振荡。

STM32 上的优先方案：

  • 用定时器作为控制节拍，明确每个环路频率。
  • ADC 使用定时器触发 + DMA，记录“本次控制使用的是哪一组采样”。
  • IMU/编码器/通信数据保存时间戳或序号；低频测量到来时只做更新，高频控制周期持续预测。
  • 发现 dt 异常大时限幅或丢弃本次计算，并记录故障计数。

4. 参数难调：Q/R 不可脱离硬件和工况

R 不能只凭感觉填写，它反映传感器在当前安装、电源、温度和振动条件下的测量波动。Q 反映模型漏掉的扰动和状态变化速度。换了电机、编码器线数、机械结构或采样率，旧参数可能不再合适。

建议建立一套可复现的调参记录：

记录项 目的
原始测量、预测值、创新量、滤波值 判断误差来自传感器还是模型
时间戳、dt、丢帧计数 发现时序问题
Q、R、初始 P、门限 让参数可追溯、可回退
电压、电流、温度、负载工况 判断参数是否只在某一种工况有效

先离线回放日志调 R/Q，再上机做小幅度阶跃和扰动测试，比在运动设备上盲调安全得多。

5. 状态不可观测：没有信息就估不出来

滤波器不能凭空创造传感器信息。例如仅靠加速度计无法可靠区分“倾斜”和“线加速度”；只测位置而没有足够时间变化或模型约束时，速度估计也可能不稳定。

处理方式不是把 Q/R 调得更复杂，而是：增加合适传感器、加入已知控制输入、限制工作工况，或承认该状态在当前硬件下不可可靠估计。做方案前应先问“这个状态能否从现有测量中被观测到”。

6. 算力和数值稳定性：MCU 不适合无节制堆矩阵

一维和二状态滤波通常足够放在 STM32 控制循环里；大状态矩阵、频繁求逆和浮点格式化不适合塞进高频 ISR。多状态算法要先测量执行时间、栈占用和最坏时延。

建议：

  • 高优先级 ISR 只做固定次数计算，日志交给 DMA/UART 低优先级任务。
  • 一维/二状态使用 float 前先确认 MCU 是否有 FPU；无 FPU 时评估频率、优化等级或定点实现。
  • 对 P、分母和输入做 isfinite/下限检查；多维实现使用 Joseph 形式保持协方差稳定。
  • 不为“看起来高级”而上 EKF。能用一维滤波解决的问题，一维通常更容易测试和维护。

7. 估计值不是安全保护，不能直接替代故障逻辑

卡尔曼输出平滑后可能掩盖突然的真实故障。电机电流突升、机械撞限位、传感器掉线时，控制逻辑必须优先看原始保护信号和故障状态。

十四、与 PID 一起调试的检查清单

  1. 串口或上位机同时画出原始测量值、滤波值、目标值、PID 输出。
  2. 检查滤波值是否有合理延迟，且没有削平真实阶跃变化。
  3. 先关掉积分项验证 P 控制，再逐步加 I，最后再考虑 D。
  4. PID 输出必须限幅；积分项需要防饱和，否则执行器到达上限后积分会继续累积。
  5. 电机系统还要检查电流限幅、堵转、电源压降和急停逻辑。滤波与 PID 都不能替代硬件保护。

十五、复习时只记住这几句话

  • Q 描述模型/过程有多不可靠，R 描述传感器有多不可靠。
  • 先预测，再在有效测量到来时更新。
  • 单位、dt、标定和异常值检查比公式本身更容易决定效果。
  • 卡尔曼给 PID 更可信的反馈，PID 根据反馈驱动执行器。
  • 先记录数据再调参数，别凭感觉同时改一堆参数。
