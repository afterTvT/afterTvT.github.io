二、最核心的类型：cv::Mat

cv::Mat 是 OpenCV 的图像和矩阵容器。它既可以表示灰度图、彩色图，也可以表示变换矩阵、特征向量。

四、颜色空间：OpenCV 默认是 BGR，不是 RGB

用 imread(..., IMREAD_COLOR) 读入的三通道图像顺序是 BGR。把它交给 Qt、OpenGL、Web 前端或某些深度学习框架时，常常需要转成 RGB。

HSV 很适合按颜色分割，因为色相、饱和度、亮度分开了；但 H 的范围不是 0 到 360，而是 0 到 179。红色跨越 HSV 范围两端，通常需要两段阈值后再合并。

八、工程里容易忽略的性能点

1. 少做不必要的 clone

clone() 会复制整张图，4K 图像的拷贝成本很高。默认优先传 const cv::Mat&，只在算法会修改输入、跨线程持有数据或需要独立结果时深拷贝。

2. 复用输出 Mat

OpenCV 大多数算子会在输出尺寸和类型一致时复用已分配内存。视频循环中把 Mat 定义在循环外即可，避免反复声明大容器和额外复制。

3. 控制调试窗口和磁盘写入

imshow、imwrite 和大量 std::cout 都可能成为实时程序瓶颈。调试完成后应通过日志等级或宏关闭这些操作，而不是留在每帧循环中。

4. 多线程前先确认数据所有权

Mat 的引用计数让“传递对象”很便宜，但不代表两个线程可以同时写同一块像素。跨线程队列中，明确约定只读、独占写入，或在入队时 clone()。

九、常见踩坑清单

  1. imread 返回空：相对路径是相对“程序工作目录”，不是相对 main.cpp。打印当前目录或改用绝对路径定位。
  2. BGR/RGB 搞反：图像整体偏蓝或偏红时，优先检查 cvtColor。
  3. Rect(x, y, width, height) 把右下角坐标当成宽高：会导致 ROI 异常甚至越界。
  4. 二值化前忘记灰度化：threshold 对多通道图的结果通常不是预期。
  5. 用 JPEG 保存掩码：有损压缩会污染二值边缘，掩码和标注优先保存 PNG。
  6. findContours 后原二值图变了：传 clone()。
  7. Release 能运行、Debug 不能运行：检查 OpenCV 的 Debug/Release 库、DLL 和运行时是否匹配。
  8. 程序在自己的电脑能跑，换机器打不开：部署时要带上 OpenCV DLL，或使用包管理器、安装包和依赖扫描工具整理运行时文件。

十、一个可复用的最小模板

后续做项目时，把每一步中间结果都保留一个可开关的调试窗口或调试图文件。视觉算法的错误往往不在最后一步，而是从输入颜色、阈值、ROI 或坐标系开始就已经偏了。先看中间结果，比盲目调最后的参数快得多。
